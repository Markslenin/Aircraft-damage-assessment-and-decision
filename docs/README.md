# Damaged Fixed-Wing Aircraft Online Identification and Decision

## Scope

This project is organized with MATLAB Project + Simulink + Aerospace Blockset and targets a staged prototype for:

`damage modeling -> residual-driven identification -> controllability/trim assessment -> mission decision`

The current milestone is **P3.5**:
- error is decomposable and diagnosable
- dataset coverage is broader and more balanced
- identifier features and training strategy are stronger
- decision logic is more robust to eta noise and low confidence
- closed-loop evaluation includes conservative vs unsafe mismatch analysis

The project now also includes a **P4-lite visualization layer**:
- visualization toolchain check
- model-side `Visualization Interface`
- demo scenario plotting/export
- architecture/program-flow/presentation diagrams
- model snapshot export for presentation use

## Project Structure

- `models/`
  Main Simulink model and subsystem skeletons.
- `scripts/`
  Initialization, dataset generation, training, benchmark, analysis, and validation entry points.
- `functions/`
  Damage semantics, nominal predictor, residual processing, feature building, identifier training/inference, assessment, and decision logic.
- `data/`
  Generated research datasets.
- `results/`
  Evaluation artifacts, CSV summaries, and figures.
- `docs/`
  Project documentation.

## Current Pipeline

```text
theta_d / plant trajectory
  -> nominal predictor
  -> sensor residuals
  -> residual filtering
  -> feature generation
  -> identifier
  -> ctrl metrics / trim assessment
  -> robust decision manager
  -> closed-loop consistency analysis
```

## Damage Vector

`theta_d` is a fixed `12 x 1` continuous damage vector, each element in `[0, 1]`.

| Index | Name | Meaning |
| --- | --- | --- |
| 1 | `left_inner_wing` | left inner wing structural damage |
| 2 | `left_outer_wing` | left outer wing structural damage |
| 3 | `right_inner_wing` | right inner wing structural damage |
| 4 | `right_outer_wing` | right outer wing structural damage |
| 5 | `left_horizontal_tail` | left horizontal tail damage |
| 6 | `right_horizontal_tail` | right horizontal tail damage |
| 7 | `vertical_tail` | vertical tail damage |
| 8 | `left_aileron_eff` | left aileron effectiveness loss |
| 9 | `right_aileron_eff` | right aileron effectiveness loss |
| 10 | `elevator_eff` | elevator effectiveness loss |
| 11 | `rudder_eff` | rudder effectiveness loss |
| 12 | `thrust_eff` | thrust effectiveness loss |

For structured outputs, the feature builder separates:

### 9. Identified vs Oracle Modes

Oracle mode:
- uses true `theta_d`
- computes oracle damage effects and control metrics
- runs trim and decision logic on oracle values

Identified mode:
- runs identifier first
- uses estimated `eta_hat` or reserved `theta_hat`
- continues through trim and decision logic with confidence-aware guarding

### 10. Closed-Loop Metrics

Closed-loop batch script:
- `scripts/run_identifier_closed_loop_batch.m`

Decision consistency script:
- `scripts/evaluate_decision_consistency.m`

Tracked metrics now include:
- average eta error
- controllability match rate
- trim feasibility match rate
- decision mode match rate
- conservative decision rate
- `conservative_correct_count`
- `conservative_overtrigger_count`
- `unsafe_undertrigger_count`
- `dangerousMismatchCount`
- mode confusion matrix

Definitions:
- conservative correct:
  identified result is more conservative than oracle, but does not create a dangerous mismatch
- conservative overtrigger:
  identified result triggers `STABILIZE`, `DIVERT`, or `EGRESS_PREP` too often relative to oracle
- unsafe undertrigger:
  identified result is more aggressive than oracle
- dangerous mismatch:
  at minimum:
  - oracle = `UNRECOVERABLE` or `EGRESS_PREP`, but identified = `RETURN`
  - oracle = `DIVERT` or `STABILIZE`, but identified = `RETURN` with high confidence

- `models/main_damaged_aircraft.slx`

`Visualization Interface` is decoupled from the control chain and accepts:
- aircraft state vector
- decision mode proxy
